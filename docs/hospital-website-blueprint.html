1. 프로젝트 개요

1.1 이 기획서의 목적

이 문서는 단순한 홈페이지 제작 매뉴얼이 아닙니다. 환자가 병원을 검색하는 순간부터 진료를 마치고 지인에게 소개하기까지, 그 전체 여정을 하나의 디지털 시스템으로 설계하기 위한 청사진입니다.

병원 홈페이지는 "있으면 좋은 것"이 아니라 "없으면 존재하지 않는 것"입니다. 2026년 현재, 환자의 87%가 병원 방문 전 온라인 검색을 하고, 그중 73%가 홈페이지를 통해 최종 결정을 내립니다. 네이버 플레이스와 카카오맵의 기본 정보만으로는 환자의 깊은 고민에 답할 수 없습니다. "이 병원이 정말 잘하는가?", "내 상황에 맞는 치료법은 무엇인가?", "비용은 얼마나 드는가?", "어떤 의사가 담당하는가?" — 이런 질문에 설득력 있게 답하는 홈페이지를 가진 병원만이 환자의 예약 버튼 클릭을 이끌어냅니다.

이 기획서는 {병원명} (예: 서울비디치과)의 홈페이지 https://{도메인} (예: bdbddc.com)을 구축하기 위한 완전한 설계도입니다. 레퍼런스 사이트인 서울비디치과에서 실제로 검증한 구조와 전략을 기반으로, 각 병원의 특성에 맞게 적용합니다. 단순히 "이런 페이지가 필요합니다"라는 목록이 아니라, 왜 그 페이지가 필요한지, 그 페이지가 환자 퍼널의 어느 단계에서 어떤 역할을 하는지, 그리고 어떻게 만들어야 실제로 전환이 일어나는지를 상세하게 설명합니다.

1.2 적용 대상

이 기획서는 {진료과} (예: 치과)를 기준으로 작성되었으며, 의료기관 전반에 적용할 수 있습니다. 치과는 치료 종류가 다양하고(임플란트부터 스케일링까지), 비급여 비율이 높아 환자의 온라인 비교 행동이 가장 활발한 진료과이기 때문에 기획서의 밀도가 자연스럽게 높아집니다. 피부과와 성형외과는 비포/애프터 갤러리와 비주얼 마케팅을 강화하면 그대로 적용됩니다. 내과와 일반의원은 치료 페이지를 진료과목 페이지로 전환하고 건강검진 예약 시스템을 추가하면 됩니다. 한의원은 한방 치료 원리 설명 콘텐츠와 체질 진단 퀴즈 같은 리드 마그넷을 접목하면 차별화됩니다. 안과와 이비인후과는 치료 항목이 상대적으로 적으므로 구조를 간소화해 적용합니다. 다만 종합병원은 진료과 수와 조직 구조가 워낙 크기 때문에 별도 설계가 필요합니다.

홈페이지의 모든 구조와 기능은 이 퍼널의 각 단계에 대응합니다. 단계별로 살펴보겠습니다.

2. 사이트 구조 설계

2.1 전체 구조 — 왜 {총페이지수}페이지 (예: 80페이지)인가

"병원 홈페이지에 80페이지가 필요한가?"라는 질문을 자주 받습니다. 답은 간단합니다. 환자의 질문이 그만큼 많기 때문입니다. "임플란트는 아프나요?", "아산에서 얼마나 걸리나요?", "{대표원장} (예: 문석준) 원장 경력이 어떻게 되나요?" — 이 모든 질문에 전용 페이지로 답하면, 각 페이지가 검색엔진에서 독립적으로 노출됩니다.

페이지 구성은 다음과 같습니다. 핵심 정보 페이지 5개(메인, 예약, 오시는 길, 비용, 의료진 목록), 치료 상세 페이지 {치료페이지수}개 (예: 24개) + 전체 목록 1개, 의사 프로필 {의사프로필수}개 (예: 15개), 지역 SEO 랜딩 페이지 {지역페이지수}개 (예: 16개), 콘텐츠/신뢰 페이지(갤러리, 칼럼, 영상, FAQ 등), 병원 정보/법적 페이지(미션, 층별 안내, 공지, 개인정보처리방침, 이용약관, 404), 회원/관리 시스템. 이 구조는 기준이며, 치료 항목이 적은 병원은 30~40페이지로 시작하고, 규모가 큰 병원은 100페이지 이상으로 확장합니다.

2.2 페이지 우선순위 — 어디에 집중할 것인가

모든 페이지의 중요도가 같지 않습니다.

레벨 1: 핵심 페이지 (5개) — 사이트의 뼈대입니다. 메인 페이지(/)는 병원의 첫인상이자 모든 트래픽의 허브입니다. 예약/상담 페이지(/reservation)는 전환의 최종 목적지입니다. 오시는 길(/directions)은 방문 결심을 굳히는 마지막 페이지입니다. 비용 안내(/pricing)는 환자가 가장 궁금해하면서도 대부분의 병원이 제대로 공개하지 않는 정보입니다 — 이것을 투명하게 공개하는 것만으로 신뢰를 크게 얻습니다. 의료진 목록(/doctors)은 "누가 나를 치료하는가?"라는 가장 본질적인 질문에 답합니다. 레벨 2: 치료/진료 페이지 — 매출에 직결됩니다. {주력치료} (예: 임플란트, 교정, 소아치과, 심미레진, 글로우네이트)가 가장 중요합니다. 치료 페이지는 {치료카테고리} (예: 전문센터 / 일반·보존 / 잇몸·외과) 체계로 분류합니다. 레벨 3: 의사 프로필 — 신뢰 구축의 핵심입니다. 환자는 "이 의사가 어디를 나왔고, 경력이 몇 년이고, 어떤 분야를 전문으로 하는지"를 반드시 확인합니다. 레벨 4: 지역 SEO 페이지 — "{도시명} {진료과}" (예: "천안 치과"), 인접 도시 검색 (예: "아산 임플란트", "세종 교정") 등 지역+치료 키워드를 잡기 위한 전략 페이지입니다. {서비스지역목록} (예: 천안, 아산, 불당, 세종, 대전, 청주, 평택, 공주, 당진, 서산, 논산, 예산, 홍성, 충주, 진천, 안성) 총 {지역페이지수}개 (예: 16개) 지역을 타겟합니다. 지역 선정 기준은 1순위가 병원 소재 도시 내 주요 동/구, 2순위가 인접 도시(차로 15~20분), 3순위가 차로 30분 거리, 4순위가 차로 1시간 거리(이 경우 고단가 전문 치료에 한해 설득력이 있습니다)입니다.

2.3 네비게이션 — 환자가 길을 잃지 않게

GNB(Global Navigation Bar)는 다음과 같이 구성합니다. 좌측에 로고, 중앙에 메인 메뉴(진료안내 드롭다운, 의료진, 미션·비전, 층별안내, 오시는 길, FAQ, 공지), 우측에 액션 버튼(전화 {전화번호} (예: 041-415-2892), 로그인/가입, 예약하기)을 배치합니다. 진료안내 드롭다운은 {치료카테고리} (예: 전문센터 / 일반·보존 / 잇몸·외과) 체계에 따라 분류합니다. 모바일 하단 고정 CTA Bar는 전화, 카카오톡, 예약하기(강조), 오시는 길 4개 버튼으로 구성합니다. 이 바는 스크롤해도 항상 화면 하단에 고정되어야 합니다. 모바일 트래픽이 전체의 75% 이상을 차지하는 상황에서, 환자가 "지금 전화하고 싶다"고 느끼는 순간 엄지손가락 하나로 터치할 수 있어야 합니다. 데스크톱 플로팅 CTA는 화면 우측 하단에 맨 위로 이동, 카카오톡 상담, 전화 상담 3개 아이콘을 세로로 배치합니다.

CTA 배치의 원칙은 "환자가 어떤 페이지의 어떤 위치에서 스크롤을 멈추더라도, 시야 안에 최소 1개의 CTA가 보여야 한다"입니다. 헤더에 전화+예약 버튼(필수), 히어로에 상담 예약+전화(필수), 본문 중간에 네이버 예약+카카오(권장), 하단에 네이버 예약+카카오+전화(필수), 모바일 고정 바에 전화+카카오+예약+길찾기(필수), 데스크톱 플로팅에 카카오+전화+맨위로(권장), 푸터에 전화+주소+소셜(필수).

3. 페이지별 상세 기획

3.1 메인 페이지 — 3초 안에 승부를 결정짓는 곳

메인 페이지는 총 13개 섹션으로 구성합니다. 순서가 중요합니다. 환자가 스크롤하면서 자연스럽게 퍼널을 따라 내려가도록 설계합니다.

첫 번째, 히어로 섹션. 운영 배지("오늘 20시까지 진료중" — {365일진료} (예: Yes)이면 "365일 진료" 병기), 서브타이틀로 "{도시명} {동네명}" (예: 천안 불당동) 명시, 메인 카피 "{메인카피}" (예: "과잉진료 없는 양심치과"), 핵심 통계 배지 — {차별점1} (예: 서울대 15인) | {차별점2} (예: 365일) | {차별점3} (예: 6개 수술실) 등, CTA 버튼(주: 상담 예약하기 / 보조: 비용 안내), 리뷰 통계 — 네이버 {네이버평점} (예: 4.85) / 구글 {구글평점} (예: 4.9).

히어로 카피라이팅 공식: 환자의 고민을 대변하는 헤드라인 + 우리가 해결하겠다는 약속. 예시(치과 기준): "다른 곳에서 안 된다고요?" → "저희가 해드리겠습니다"(임플란트), "교정은 아프고 티나잖아요" → "투명하게, 편안하게"(교정), "우리 아이가 치과를 무서워해요" → "놀이처럼 즐거운 진료"(소아치과).

두 번째, 운영 상태 배지. 세 번째, 핵심 차별점 카드 — {차별점1} (예: 서울대 출신 15인 원장), {차별점2} (예: 365일 진료), {차별점3} (예: 6개 독립 수술실), {차별점4} (예: 원내 기공소), {차별점5} (예: 디지털 정밀 진단), {차별점6} (예: 에어샤워 감염관리). 네 번째~열세 번째: 주력 치료 소개(4~6개), 의료진 소개({대표원장} (예: 문석준) + 3~5명), Before/After 슬라이더, 구글/네이버 리뷰 위젯, 실시간 구글 리뷰 API, 비용 미리보기, FAQ(3~5개), 최신 칼럼/영상, 최종 CTA, 푸터.

▶ 실전 예시: 서울비디치과 메인 페이지 13개 섹션 전체 구성

  1. 히어로: 배경 — 병원 내부 고화질 사진(400평 인테리어). 메인 카피 "과잉진료 없는 양심치과". 배지 — "서울대 15인 | 365일 진료 | 6개 수술실". CTA "무료 상담 예약하기"(골드 버튼) + "비용 안내"(화이트 버튼). 좌측 하단 리뷰: 네이버 ★4.85 / 구글 ★4.9.
  1. 운영 상태: "오늘(수요일) 20시까지 진료중" — 실시간 시간 계산, 현재 시각 기준 남은 시간 표시. 주말이면 "토요일 17시까지 진료".
  1. 차별점 카드 6장: 각각 아이콘 + 제목 + 설명 1줄. ① 서울대 15인 원장 — "복잡한 케이스도 다학제 협진" ② 365일 진료 — "일요일·공휴일도 오전 10시~오후 4시" ③ 6개 독립 수술실 — "1인 1수술실, 교차감염 제로" ④ 원내 기공소 — "보철물 외주 없이 정밀 제작" ⑤ 디지털 정밀 진단 — "CT → 3D 시뮬레이션 → 네비게이션" ⑥ 에어샤워 — "수술실 진입 전 공기 청정 시스템"
  1. 주력 치료 5종: 임플란트(카드: "다른 곳에서 안 된다고요?" + 이미지 + CTA), 교정("티 안 나게, 편안하게"), 소아치과("우리 아이 첫 치과, 무섭지 않아요"), 심미레진("자신 있게 웃으세요"), 글로우네이트("서울비디치과 자체 개발 프리미엄 심미 솔루션")
  1. 의료진 소개: 문석준 대표원장(서울대·치과겁통령) + 주요 원장 4명. 각각 사진 + 이름 + 전문분야 1줄 + "프로필 보기" 링크. 하단 "전체 15명 의료진 보기 →"
  1. Before/After 슬라이더: 최신 3개 케이스. 좌우 슬라이드 비교. "갤러리 전체 보기 →"
  1. 구글 리뷰 실시간 위젯: API로 최신 5~10개 리뷰 표시. 별점 + 작성자 이니셜 + 리뷰 내용 일부. "구글 리뷰 전체 보기 →"
  1. 비용 미리보기: 주요 3~5개 치료의 가격 범위 테이블. "전체 비용 보기 →"
  1. FAQ 3개: "예약은 어떻게 하나요?", "일요일에도 진료하나요?", "주차 가능한가요?"
  1. 최신 칼럼/영상: 블로그 RSS 최신 3건 + 유튜브 최신 2건 썸네일.
  1. 최종 CTA: "지금 바로 상담 예약하세요" + 전화/카카오/네이버 3종 버튼 대형 배치.
  1. 오시는 길 미니맵: 지도 + "충남 천안시 서북구 불당25로 60, 8~10층" + "길찾기 →"
  1. 푸터: 병원명, 대표원장, 사업자번호, 주소, 전화, 진료시간, 소셜링크, ©, 개인정보/이용약관 링크.

3.2 치료 상세 페이지 — 환자의 불안을 지식으로 바꾸는 공간

{치료페이지수}개 (예: 24개) 모든 치료 페이지는 동일한 7단 구조를 따릅니다.
  1. 히어로 — 감성 카피 + 치료 정의 설명박스. 2. 고민 공감 — "혹시 이런 고민이 있으신가요?" 3~4가지. 3. 치료 종류 — 세부 옵션 카드. 4. 차별점 — "왜 {병원명} (예: 서울비디치과)에서?" 해당 치료에 맞는 4~6개. 5. 치료 과정(HowTo) — 단계별 설명 → Schema.org HowTo 마크업 → 구글 리치스니펫. 6. 담당 의료진 — 해당 치료 전문 의사 2~3명 + 프로필 링크. 7. FAQ — 5~8개 질문 → Schema.org FAQPage 마크업 → 구글 FAQ 리치스니펫.
