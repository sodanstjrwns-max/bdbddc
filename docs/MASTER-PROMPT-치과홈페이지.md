> 2. 붙여넣은 신청서 + 이 마스터 프롬프트 전체를 Genspark(Super Agent / AI Developer)에 제출한다.

## §1. 최우선 원칙 (Non-negotiable)

1. **디자인 품질:** 결과물은 **"서울비디치과(bdbddc.com)를 능가하는 2026 최신 하이엔드 인터랙티브 홈페이지"** 여야 한다.
   "병원 홈페이지 같지 않게, 럭셔리 매거진 + 테크 스타트업 수준"으로. **"우와"가 나와야 한다.**
2. **디자인은 매번 압도적으로 다르게:** 비디치과의 색·폰트·레이아웃을 **복제 금지**. §0의 A 입력을 반영하고, 나머지는 §6 원칙으로 독창적으로 새로 설계.
3. **SEO/AEO가 제작의 최대 목적:** §5의 기술 골격은 **빠짐없이 전부** 구현.
4. **의료광고법 준수:** §8을 100% 지킨다. 별점·평점·후기 단정·"최고/1위" 등 과장·보증 표현 금지.
5. **모든 페이지가 서로 링크로 연결**(허브-스포크): 진료↔담당원장↔비포애프터가 양방향 연결. 고아 페이지 0건.
6. **데이터는 Cloudflare(R2/D1) + GitHub 자동배포**, 모든 콘텐츠 크롤링 가능.

## §2. 사이트 구조 (약 100페이지 골격 — 비디치과 검증 구조)

### A. 핵심 페이지
- `/` **메인** — 히어로 + 병원 강점 + 진료 요약 + 의료진 + 비포애프터 미리보기 + 위치/예약 CTA
- `/mission` **병원 미션** — *히어로급으로 화려하게* (풀스크린, 시네마틱, 스크롤 스토리텔링)

### B. 의료진 (`/doctors/`)
- `/doctors/` 전체 의료진 소개 (그리드/카드)
- `/doctors/{원장슬러그}` 원장별 개별 페이지 — 학력·전공·면허·경력·논문·**주력 진료 링크**·**해당 원장 비포애프터 링크**
- (선택) 진료과별 의료진 페이지

### C. 진료과목 (`/treatments/`)
- `/treatments/` 전체 진료 허브
- **핵심 진료 3가지** = 초상세 페이지(최소 1,500자+, FAQ 20개+, 비교표, 단계 설명, 비용 안내, 스키마)
- **그 외 모든 진료** = 제법 볼륨 있는 소개 페이지(각 진료가 독립 페이지)
- 각 진료 페이지 → 담당원장 / 관련 비포애프터 / 관련 백과사전 용어로 인링크
- (SEO 확장) 시술 세분화 페이지(예: 임플란트→수면/즉시/네비게이션…)

### D. 콘텐츠
- `/cases/` **비포애프터** — §3 상세 스펙 참조
- `/blog/` **블로그** — §4 에디터 스펙 참조
- `/notice/` **공지사항** — 대표(상단고정) 공지 지정 가능, 사진 업로드
- `/encyclopedia/` **백과사전** — **500개 이상** 용어, 진료/비포애프터와 인링크 자동 형성
- `/faq/` **자주묻는질문** — 진료별 FAQ를 모아 한 곳에서 + 진료별 개별 FAQ 페이지

### E. 내원 안내
- `/directions` 오시는 길 (지도·대중교통·주차)
- `/hours` 진료시간 안내
- `/pricing` 수가(비급여 진료비) 안내 — 보건복지부 고시 준수

### F. 지역 SEO 페이지 (`/area/`) — **제작의 핵심 목적**
- §0의 타겟 지역마다 `/area/{지역}` + `/area/{지역}-{핵심진료}` 변형 페이지 생성
- 각 페이지: 해당 지역명 + 진료 키워드 최적화, LocalBusiness/지역 스키마, 인근 지역 인링크

### G. 회원/예약/기타
- `/auth/register` 회원가입 (§7), `/auth/login`, `/auth/mypage`
- `/reservation` 예약, `/reservation/thank-you`
- `/careers` 채용 (선택)
- `/privacy` 개인정보처리방침, `/terms` 이용약관
- `/404` 커스텀 404

### H. 관리자 (`/admin/`) — §9
- 대시보드 / 회원관리 / 비포애프터 / 블로그 / 공지 / (예약) — 작성·수정·삭제·조회수

## §3. 비포애프터(케이스) 상세 스펙

- **사진 업로드 4종:** 파노라마 전(前)/후(後), 구내포토 전/후 → **업로드 안 된 사진은 화면에 표시 안 함**
- **슬라이드 비교 뷰어:** 전/후 사진을 슬라이더로 비교(before-after slider)
- **메타 입력(모두 SEO 신호로 활용):**
  케이스 설명 / 환자 나이대 / 성별 / 진료 카테고리 / 지역 카테고리 / 담당 원장 / 치료 기간
- **지역 카테고리 자동완성:** 쇼핑몰 주소검색 방식.
  예) "초지" 입력 → "안산시 상록구 초지동", "안산" 입력 → "안산시" 로 입력되게(행정구역 매칭).
- **로그인 게이팅:** 비포애프터 카드 자체는 비로그인도 노출되나, **애프터(후) 사진은 로그인해야만** 보임.
- **연결:** 케이스 ↔ 담당 원장 페이지 ↔ 관련 진료 페이지 양방향 링크. 관련 백과사전 용어 인링크.
- **스키마:** ImageObject + (가능 시) MedicalProcedure 연계.

> 비디치과 검증 패턴. **모든 페이지**에 적용.

### 5-1. 메타 (페이지별 고유)
- `<title>` 30~60자, 핵심 키워드 앞쪽 + ` | {병원명}` 일관 배치, **중복 0건**
- `meta description` 120~160자 + CTA(전화번호 등)
- `meta keywords`, `author`, `robots`(index,follow,max-image-preview:large,max-snippet:-1)
- **Geo/Local:** `geo.region`, `geo.placename`, `geo.position`, `ICBM`(좌표는 §0 주소 기반)
- **AEO 전용 메타(★ 차별점):** `subject`, `abstract`, `ai-summary` — LLM이 병원을 정확히 요약하도록 핵심 사실(위치·진료·전화·강점) 명시
- **E-E-A-T 메타:** `medical-content`(의료 정보 감수 표기)
- **Open Graph 전체:** og:title/description/type/url/locale/site_name/image(1200×630)/image:alt
- **Twitter Card:** summary_large_image 세트
- 검색엔진 인증: `google-site-verification`, `naver-site-verification`(발급 후 삽입)

### 5-2. 구조화 데이터 (JSON-LD) — 비디치과는 30종+ 사용
- **Dentist / MedicalBusiness / LocalBusiness** (주소·전화·좌표·OpeningHoursSpecification)
- **Organization** + **WebSite**(SearchAction 포함)
- **Person** (원장 1인당 1개)
- **MedicalProcedure** (시술별 — 임플란트/교정/라미네이트 등 각각)
- **FAQPage** (Question/Answer — FAQ 페이지마다)
- **BreadcrumbList** (전 페이지)
- **Article/BlogPosting** (블로그)
- **ImageObject / VideoObject** (영상 임베드 시)
- **City / AdministrativeArea** (지역 SEO 페이지)
- **SpeakableSpecification** (음성검색)
- ⚠️ **Review/AggregateRating(별점)은 의료광고법 검토 전까지 넣지 않음** — §8 참조
- Rich Results Test 통과 필수

### 5-3. 캐노니컬 / 인덱싱 / hreflang
- 모든 페이지 `canonical` 절대경로
- 다국어 시 `hreflang`(ko/en/jp/cn… + x-default) 상호 연결
- `sitemap.xml`(분할: main / area / encyclopedia / intl) + lastmod 정확 → GSC·네이버 제출
- `robots.txt` 적절 설정(불필요 페이지 차단, AI 크롤러 GPTBot/ClaudeBot/PerplexityBot **허용**)
- `llms.txt` 작성(AI 친화 사이트 요약)
- `feed.xml`(RSS — 빠른 인덱싱)

### 5-4. URL 구조
- 영문 슬러그 통일, 75자 이하, 키워드 포함(예: `/area/cheonan-implant`)
- 위계 3depth 이하, 끝 슬래시 통일, www/non-www 301 통일

### 5-5. 성능 / Core Web Vitals
- LCP ≤2.5s, INP ≤200ms, CLS ≤0.1, PSI 모바일 80+↑
- 이미지 WebP/AVIF + lazy-load + 200KB 이하 + descriptive 파일명 + 전부 alt
- 폰트 preload + preconnect(fonts/cdn), CSS critical 인라인, minify, Brotli/GZIP, Cloudflare CDN
- HTTPS + HSTS, 혼합콘텐츠 0

### 5-6. 온페이지
- H1 페이지당 1개(핵심 키워드 포함), H2~H6 위계 정확(skip 금지)
- 시술 페이지 1,500자+, thin content 0, 표·리스트·이미지로 가독성
- 내부링크 허브-스포크, 앵커텍스트 키워드 자연 포함, 깨진 링크 0, 고아 페이지 0

### 5-7. AEO (LLM 검색 최적화)
- 질문형 H2/H3 + **답변 즉시 제공**(질문 직후 1~2문장 핵심) → 아래 상세
- 명확한 정의·숫자·기간, 비교표, 단계형(1·2·3), 결론 요약 박스
- 롱테일/PAA 질문 키워드 콘텐츠화, 구어체 음성검색 대응

### 5-8. 분석 도구 셋업(스크립트 슬롯 — §0 값으로 치환)
- Google Tag Manager / GA4 / (선택) Amplitude / Meta Pixel / 네이버 애널리틱스
- GSC·네이버 서치어드바이저·Bing Webmaster 등록 안내 포함

## §6. 디자인 원칙 (A+B 하이브리드 — "비디치과를 능가하라")

### 6-1. 절대 규칙
- §0의 **디자인 요구사항(A)** 은 **반드시** 반영(메인컬러·무드·폰트·히어로 컨셉 등).
- §0에서 비운 항목만 **AI가 병원 성격에 맞춰 2026 트렌드로 독창적으로 설계(B)**.
- **비디치과의 구체적 비주얼(브라운 계열·특정 폰트·특정 레이아웃) 복제 금지.** 매 프로젝트 = 새 디자인 언어.
- ⚠️ **Q27(참고 URL) 함정 주의:** 원장님 다수가 참고 사이트로 "비디치과"를 적는다. 이는 **"비디치과 수준의 완성도·SEO·신뢰감을 원한다"는 뜻이지 "비디치과처럼 생기게 해달라"가 아니다.** 골격·품질만 벤치마크하고 **디자인은 Q24·Q25 기반으로 완전히 다르게** 만든다. 같은 기수에서 여러 병원이 비디치과를 참고로 적어도 결과물은 서로 닮으면 안 된다.

### 6-2. 품질 기준 ("우와" 레벨)
- 럭셔리 매거진 + 하이엔드 테크 브랜드 수준. "병원 홈페이지 같지 않게."
- 깔끔하되 과감하게: 충분한 여백, 정교한 그리드, 강렬한 타이포그래피 위계.
- **폰트 세트 구성에 특히 신경**: 헤드라인/본문/포인트 폰트의 조화로운 페어링.

### 6-3. 인터랙션 / 모션 (비디치과 초월의 핵심)
- 2026 트렌드: 스크롤 연동 애니메이션(scroll-driven), 패럴랙스, 부드러운 페이지 전환,
  마이크로 인터랙션(호버·커서·버튼), 섹션 진입 reveal, 시네마틱 히어로.
- **단, 성능(Core Web Vitals)과 충돌 금지** — 모션은 GPU 효율적으로, 모바일에서도 60fps.
- 라이브러리 예: GSAP / Lenis(부드러운 스크롤) / Framer Motion 등 적절히. 과하지 않게, 의도적으로.

### 6-4. 이미지 / 레이아웃
- 사진은 "뭉텅이로 나열 금지." **럭셔리 매거진처럼** 각 사진을 최적 위치·최적 레이아웃으로 배치.
- 파비콘 / OG 이미지 / 애플터치아이콘을 병원 아이덴티티에 맞게 생성·적용.

### 6-5. 신뢰 + 혁신 균형
- 실험적·혁신적이되 의료기관의 **신뢰감**을 해치지 않기. 고급스러움 유지.

### 6-6. 반응형 / 접근성
- 완전 반응형. 모바일 폰트 16px+, 터치타겟 48px+, 가로스크롤 0.
- WCAG 2.1 AA: 색상대비 4.5:1+, 키보드 내비, 스크린리더 호환.
