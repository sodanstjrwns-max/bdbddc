# 1. 프로젝트 개요

## 1.1 이 기획서의 목적

이 문서는 단순한 홈페이지 제작 매뉴얼이 아닙니다. 환자가 병원을 검색하는 순간부터 진료를 마치고 지인에게 소개하기까지, 그 전체 여정을 하나의 디지털 시스템으로 설계하기 위한 청사진입니다.

병원 홈페이지는 "있으면 좋은 것"이 아니라 "없으면 존재하지 않는 것"입니다. 2026년 현재, 환자의 87%가 병원 방문 전 온라인 검색을 하고, 그중 73%가 홈페이지를 통해 최종 결정을 내립니다. 네이버 플레이스와 카카오맵의 기본 정보만으로는 환자의 깊은 고민에 답할 수 없습니다. "이 병원이 정말 잘하는가?", "내 상황에 맞는 치료법은 무엇인가?", "비용은 얼마나 드는가?", "어떤 의사가 담당하는가?" — 이런 질문에 설득력 있게 답하는 홈페이지를 가진 병원만이 환자의 예약 버튼 클릭을 이끌어냅니다.

이 기획서는 서울비디치과(bdbddc.com)를 실제로 구축하면서 검증한 모든 기능, 구조, 전략을 담고 있습니다. 단순히 "이런 페이지가 필요합니다"라는 목록이 아니라, 왜 그 페이지가 필요한지, 그 페이지가 환자 퍼널의 어느 단계에서 어떤 역할을 하는지, 그리고 어떻게 만들어야 실제로 전환이 일어나는지를 상세하게 설명합니다.

## 1.2 적용 대상

이 기획서는 치과를 기준으로 최적화되어 있지만, 의료기관 전반에 적용할 수 있습니다. 치과는 치료 종류가 다양하고(임플란트부터 스케일링까지), 비급여 비율이 높아 환자의 온라인 비교 행동이 가장 활발한 진료과이기 때문에 기획서의 밀도가 자연스럽게 높아집니다. 피부과와 성형외과는 비포/애프터 갤러리와 비주얼 마케팅을 강화하면 그대로 적용됩니다. 내과와 일반의원은 치료 페이지를 진료과목 페이지로 전환하고 건강검진 예약 시스템을 추가하면 됩니다. 한의원은 한방 치료 원리 설명 콘텐츠와 체질 진단 퀴즈 같은 리드 마그넷을 접목하면 차별화됩니다. 안과와 이비인후과는 치료 항목이 상대적으로 적으므로 구조를 간소화해 적용합니다. 다만 종합병원은 진료과 수와 조직 구조가 워낙 크기 때문에 별도 설계가 필요합니다.

## 1.3 핵심 철학: 환자 퍼널 10단계

> **"환자가 병원을 인지하는 순간부터 지인에게 소개하기까지의 전체 여정을 설계한다"**
> — Patient Funnel (페이션트 퍼널)

홈페이지의 모든 구조와 기능은 이 퍼널의 각 단계에 대응합니다. 단계별로 살펴보겠습니다.

**1단계: 인지** — 환자는 "천안 임플란트", "불당동 치과" 같은 검색어로 병원을 발견합니다. 이 단계에서 홈페이지가 해야 할 일은 검색엔진에서 노출되는 것입니다. 지역 SEO 페이지 16개, 치료 페이지 24개, 134개의 구조화 데이터가 이 단계를 담당합니다.

**2단계: 관심** — 검색 결과를 클릭해 홈페이지에 들어온 순간, 첫 3초가 승부입니다. 히어로 섹션의 메인 카피, 운영 상태 배지("오늘 20시까지 진료중"), 핵심 통계("서울대 15인 원장 · 365일 진료 · 6개 수술실")가 환자의 이탈을 막습니다.

**3단계: 탐색** — 관심을 가진 환자는 자신에게 해당하는 치료 정보를 찾습니다. 24개의 치료 상세 페이지가 "이 증상이면 이런 치료를 합니다"라는 맞춤형 답변을 제공합니다. 각 페이지에는 고민 공감 섹션, 치료 종류, 과정 설명, FAQ가 포함됩니다.

**4단계: 비교** — 환자는 반드시 다른 병원과 비교합니다. "왜 이 병원이어야 하는가?"에 답하기 위해 의사 프로필 15개(학력, 경력, 전문 분야, 소개 영상), 시설 안내(층별 가이드), 차별점 카드(디지털 장비, 감염관리, 원내 기공소)가 준비되어야 합니다.

**5단계: 신뢰** — 비교를 마친 환자는 "정말 잘하는가?"를 확인합니다. 실시간 구글 리뷰 위젯(API 연동), 비포/애프터 갤러리(슬라이더 비교), 외부 리뷰 플랫폼 링크가 사회적 증거를 제공합니다.

**6단계: 결심** — 신뢰가 쌓인 환자는 예약을 결심합니다. 이 순간을 놓치지 않기 위해 모든 페이지에 CTA(전화, 카카오톡, 네이버 예약, 온라인 폼)가 배치됩니다. 모바일 하단 고정 바, 데스크톱 플로팅 버튼, 페이지 내 중간/하단 CTA 섹션이 항상 손 닿는 곳에 있어야 합니다.

**7단계: 방문** — 예약한 환자가 실제로 찾아오게 하는 것도 홈페이지의 역할입니다. 오시는 길 페이지에 자가용/대중교통/주차 정보, 구글맵·네이버맵·카카오맵 연동, 건물 사진이 포함됩니다.

**8단계: 경험** — 실제 진료는 오프라인 영역이지만, 대기실에서 "우리 병원 Wi-Fi 연결" 안내, 모바일 예약 확인 등 디지털 접점이 경험을 보완합니다.

**9단계: 만족** — 진료를 마친 환자에게 리뷰 작성 유도 링크(구글/네이버/카카오)를 안내합니다. QR코드를 통한 리뷰 페이지 연결도 효과적입니다.

**10단계: 소개** — 만족한 환자가 지인에게 추천할 때, 공유하기 쉬운 URL 구조, Open Graph 이미지(1200×630), 소셜 공유 버튼이 바이럴을 촉진합니다.

# 2. 사이트 구조 설계

## 2.1 전체 구조 — 왜 80페이지인가

"병원 홈페이지에 80페이지가 필요한가?"라는 질문을 자주 받습니다. 답은 간단합니다. 환자의 질문이 80가지 이상이기 때문입니다. "임플란트는 아프나요?", "아산에서 얼마나 걸리나요?", "문석준 원장 경력이 어떻게 되나요?" — 이 모든 질문에 전용 페이지로 답하면, 각 페이지가 검색엔진에서 독립적으로 노출됩니다. 하나의 메인 페이지에 모든 정보를 쑤셔넣으면 "천안 임플란트"를 검색한 환자에게도, "세종 소아치과"를 검색한 환자에게도 똑같은 페이지가 뜨는데, 이것은 환자가 원하는 답이 아닙니다.

80페이지의 구성은 다음과 같습니다. 핵심 정보 페이지 5개(메인, 예약, 오시는 길, 비용, 의료진 목록), 치료 상세 페이지 25개(24개 치료 + 전체 목록), 의사 프로필 15개, 지역 SEO 랜딩 페이지 16개, 콘텐츠/신뢰 페이지 6개(갤러리, 칼럼, 영상, FAQ, 치료별 FAQ), 병원 정보/법적 페이지 7개(미션, 층별 안내, 공지, 개인정보처리방침, 이용약관, 404), 회원/관리 시스템 6개(로그인, 회원가입, 마이페이지, 관리자 3종). 이 구조는 최소 기준입니다. 치료 항목이 적은 병원은 30~40페이지로 시작하고, 규모가 큰 병원은 100페이지 이상으로 확장합니다.

## 2.2 페이지 우선순위 — 어디에 집중할 것인가

모든 페이지의 중요도가 같지 않습니다. 레벨 1에 해당하는 핵심 페이지 5개는 사이트의 뼈대입니다. 메인 페이지(/)는 병원의 첫인상이자 모든 트래픽의 허브입니다. 예약/상담 페이지(/reservation)는 전환의 최종 목적지입니다. 오시는 길(/directions)은 방문 결심을 굳히는 마지막 페이지입니다. 비용 안내(/pricing)는 환자가 가장 궁금해하면서도 대부분의 병원이 제대로 공개하지 않는 정보입니다 — 이것을 투명하게 공개하는 것만으로 신뢰를 크게 얻습니다. 의료진 목록(/doctors)은 "누가 나를 치료하는가?"라는 가장 본질적인 질문에 답합니다.

레벨 2인 치료 페이지는 매출에 직결됩니다. 환자가 "임플란트 잘하는 치과"를 검색했을 때, 임플란트 전용 페이지가 없으면 경쟁에서 밀립니다. 서울비디치과 기준으로 전문센터(고수익) 치료 5종 — 임플란트, 인비절라인(교정), 소아치과, 심미치료, 글로우네이트(자체 브랜드 미백) — 이 가장 중요합니다. 다음으로 일반/보존 진료 6종(충치, 레진, 크라운, 인레이, 신경치료, 미백), 잇몸/외과 6종(스케일링, 잇몸, 치주염, 사랑니, 턱관절, 이갈이), 확장 분야(재신경치료, 치근단절제술, 보철, 틀니, 응급, 예방)가 뒤를 잇습니다.

레벨 3인 의사 프로필은 신뢰 구축의 핵심입니다. 환자는 "이 의사가 어디를 나왔고, 경력이 몇 년이고, 어떤 분야를 전문으로 하는지"를 반드시 확인합니다. 각 의사 페이지에는 프로필 사진(전문 촬영), 학력/자격, 경력, 전문 분야, 소속 학회, 소개 영상, 진료 모습 사진, 담당 치료 링크가 포함되어야 합니다.

레벨 4인 지역 SEO 페이지는 "천안 치과", "아산 임플란트", "세종 교정" 등 지역+치료 키워드를 잡기 위한 전략 페이지입니다. 서울비디치과는 천안, 아산, 불당, 세종, 대전, 청주, 평택, 공주, 당진, 서산, 논산, 예산, 홍성, 충주, 진천, 안성 등 16개 지역을 타겟합니다. 각 페이지는 "{지역}에서 서울비디치과까지 차로 {N}분"이라는 히어로로 시작하여, 해당 지역에서 오는 교통 안내, 실제 해당 지역 환자 리뷰, 지도를 포함합니다. 지역 선정 기준은 1순위가 병원 소재 도시 내 주요 동/구, 2순위가 인접 도시(차로 15~20분), 3순위가 차로 30분 거리, 4순위가 차로 1시간 거리(이 경우 임플란트나 교정 같은 고단가 전문 치료에 한해 설득력이 있습니다)입니다.

## 2.3 네비게이션 — 환자가 길을 잃지 않게

80페이지를 가진 사이트에서 환자가 원하는 정보를 3클릭 이내에 찾게 하려면 네비게이션 설계가 결정적입니다.

**GNB(Global Navigation Bar)**는 다음과 같이 구성합니다. 좌측에 로고, 중앙에 메인 메뉴(진료안내 드롭다운, 의료진, 미션·비전, 층별안내, 오시는 길, FAQ, 공지), 우측에 액션 버튼(전화, 로그인/가입, 예약하기)을 배치합니다. 진료안내 드롭다운은 3개 카테고리로 나눕니다 — 전문센터(글로우네이트, 임플란트, 교정, 소아치과, 심미치료), 일반/보존(충치, 레진, 크라운, 인레이, 신경치료, 미백), 잇몸/외과(스케일링, 잇몸, 치주염, 사랑니, 턱관절, 이갈이). 이 구조가 중요한 이유는 환자가 "나는 잇몸이 아픈데 어디를 봐야 하지?"라고 생각할 때, 직관적으로 "잇몸/외과" 카테고리를 찾을 수 있기 때문입니다.

**모바일 하단 고정 CTA Bar**는 전화, 카카오톡, 예약하기(강조), 오시는 길 4개 버튼으로 구성합니다. 이 바는 스크롤해도 항상 화면 하단에 고정되어야 합니다. 모바일 트래픽이 전체의 75% 이상을 차지하는 상황에서, 환자가 "지금 전화하고 싶다"고 느끼는 순간 엄지손가락 하나로 터치할 수 있어야 합니다.

**데스크톱 플로팅 CTA**는 화면 우측 하단에 맨 위로 이동, 카카오톡 상담, 전화 상담 3개 아이콘을 세로로 배치합니다.

CTA 배치의 원칙은 "환자가 어떤 페이지의 어떤 위치에서 스크롤을 멈추더라도, 시야 안에 최소 1개의 CTA가 보여야 한다"입니다. 구체적으로 헤더에 전화+예약 버튼(필수), 히어로 섹션에 상담 예약+전화(필수), 본문 중간에 네이버 예약+카카오(권장), 하단 CTA 섹션에 네이버 예약+카카오+전화(필수), 모바일 하단 고정 바에 전화+카카오+예약+길찾기(필수), 데스크톱 플로팅에 카카오+전화+맨위로(권장), 푸터에 전화+주소+소셜(필수)을 배치합니다.

# 3. 페이지별 상세 기획

## 3.1 메인 페이지 — 3초 안에 승부를 결정짓는 곳

메인 페이지는 환자가 병원 홈페이지에서 가장 처음 보는 화면이자, 전체 사이트에서 가장 많은 트래픽을 받는 페이지입니다. 이 페이지의 목표는 단 하나 — 환자가 "여기 괜찮겠는데?"라고 느끼고 다음 행동(치료 페이지 클릭, 예약, 전화)으로 넘어가게 하는 것입니다.

메인 페이지는 총 13개 섹션으로 구성합니다. 순서가 중요합니다. 환자가 스크롤하면서 자연스럽게 퍼널을 따라 내려가도록 설계합니다.

**첫 번째, 히어로 섹션**은 페이지의 얼굴입니다. 운영 배지("오늘 20시까지 진료중")는 실시간성을 줘서 긴박감을 만듭니다. 환자가 "지금 전화해도 되는 곳이구나"라고 즉각 인지합니다. 서브타이틀로 지역(예: 천안 불당동)을 명시하고, 메인 카피는 병원의 핵심 가치를 한 문장으로 압축합니다(예: "과잉진료 없는 양심치과"). 그 아래에 핵심 통계 배지(서울대 15인 | 365일 | 야간 20시 | 6개 수술실)를 배치해 한눈에 차별점을 전달합니다. CTA 버튼은 주 CTA(상담 예약하기)와 보조 CTA(비용 안내)를 나란히 놓고, 최하단에 리뷰 통계(네이버 4.85 / 구글 4.9 / 만족도 98%)를 넣어 사회적 증거를 즉시 제공합니다.

히어로 카피라이팅의 공식이 있습니다. 환자의 고민을 대변하는 헤드라인 + 우리가 해결하겠다는 약속으로 구성합니다. 예를 들어 "다른 곳에서 안 된다고요?" → "저희가 해드리겠습니다"(임플란트), "교정은 아프고 티나잖아요" → "투명하게, 편안하게"(교정), "우리 아이가 치과를 무서워해요" → "놀이처럼 즐거운 진료"(소아치과). 이 공식은 메인 페이지뿐 아니라 모든 치료 페이지 히어로에도 동일하게 적용합니다.

**두 번째, 운영 상태 배지** — "오늘 20시까지 진료" 등 현재 상태를 보여주는 작은 배지입니다. 이것의 효과는 "이 사이트는 살아있는 사이트구나, 관리되고 있구나"라는 인상을 줍니다. 환자가 폐업했거나 방치된 홈페이지를 만나는 경험이 생각보다 많기 때문에, 이 작은 요소가 의외로 큰 신뢰를 만듭니다.

**세 번째, 핵심 차별점 카드** — 3~6개의 키워드를 카드 형태로 보여줍니다. "서울대 출신 15인 원장", "365일 진료", "6개 독립 수술실", "원내 기공소", "디지털 정밀 진단", "철저한 감염관리" 등입니다. 각 카드는 아이콘 + 숫자/키워드 + 한 줄 설명으로 구성합니다. 환자가 이 섹션을 훑어보는 데 5초면 충분하고, 5초 만에 "여기는 규모가 있고, 전문적이고, 꼼꼼한 병원이구나"라는 인상을 심습니다.

**네 번째부터 열세 번째까지**는 주력 치료 소개(4~6개 카드), 의료진 소개(대표원장 + 3~5명), Before/After 슬라이더, 구글/네이버 리뷰 위젯, 실시간 구글 리뷰 API 연동, 비용 미리보기, FAQ 미리보기(3~5개), 최신 칼럼/영상, 최종 CTA("지금 바로 상담 받아보세요"), 푸터 순서로 이어집니다. 각 섹션은 환자 퍼널의 관심→탐색→비교→신뢰→결심 순서를 따릅니다.

## 3.2 치료 상세 페이지 — 환자의 불안을 지식으로 바꾸는 공간

치료 페이지는 사이트에서 가장 중요한 수익 직결 페이지입니다. 24개 모든 치료 페이지는 동일한 구조를 따르되, 내용과 톤은 각 치료의 특성에 맞게 달라집니다.

첫 번째 섹션은 히어로입니다. 감성적인 카피로 시작하되, 바로 아래에 치료 정의 설명박스("임플란트란 인공 치근을 턱뼈에 심어 상실된 치아를 대체하는 치료입니다")를 배치합니다. 환자의 감정에 공감하면서도 즉시 정확한 정보를 제공하는 것이 핵심입니다.

두 번째 섹션은 고민 공감입니다. "혹시 이런 고민이 있으신가요?"라는 질문과 함께 3~4가지 일상적인 고민을 나열합니다. 임플란트 페이지라면 "다른 병원에서 뼈가 부족하다고 했다", "수술이 무섭다", "비용이 걱정된다", "오래 걸릴까 봐 망설여진다" 같은 것들입니다. 이 섹션의 효과는 "아, 여기 사람들이 내 마음을 정확히 알고 있구나"라는 공감대 형성입니다.

세 번째 섹션은 치료 종류입니다. 하나의 치료 안에도 여러 세부 옵션이 있음을 보여줍니다. 임플란트의 경우 수면 임플란트, 비절개 임플란트, 네비게이션 임플란트, 고난도 임플란트, 재수술, 공휴일 수술 등을 카드 형태로 나열합니다. 이것은 "여기는 단순히 임플란트를 하는 곳이 아니라, 내 상황에 맞는 최적의 방법을 찾아주는 곳"이라는 전문성을 어필합니다.

네 번째 섹션은 차별점입니다. "왜 이 병원에서 치료받아야 하는가?"에 대한 직접적인 답변입니다. 4~6개 포인트로 구성하며, 서울비디치과의 경우 "15인 협진 시스템", "6개 독립 수술실", "디지털 정밀 수술(0.1mm 단위)", "원내 기공소", "철저한 감염관리(에어샤워)", "365일 수술 가능"을 내세웁니다.

다섯 번째 섹션은 치료 과정(HowTo)입니다. 환자가 가장 불안해하는 것 중 하나가 "뭘 하는지 모르겠다"는 것입니다. 치료 과정을 단계별로(1. 상담 및 검사 → 2. 디지털 시뮬레이션 → 3. 식립 수술 → 4. 치유 기간 → 5. 보철물 장착 → 6. 정기 검진) 명확하게 보여주면 불안이 크게 줄어듭니다. 이 섹션은 Schema.org의 HowTo 스키마로 마크업되어 구글 검색 결과에서 단계별 리치스니펫으로 노출될 수 있습니다.

여섯 번째 섹션은 담당 의료진입니다. 해당 치료를 전문으로 하는 의사 2~3명의 사진, 이름, 전문분야, 한 줄 소개를 보여주고, 각 의사 프로필 페이지로 링크합니다. 이 연결이 중요한 이유는 "임플란트를 잘하는 사람이 구체적으로 누구인가?"라는 질문에 즉시 답하기 때문입니다.

일곱 번째 섹션은 FAQ입니다. 치료별로 5~8개의 자주 묻는 질문을 아코디언 형태로 배치합니다. "수술 시간은 얼마나 걸리나요?", "통증은 어느 정도인가요?", "보험 적용이 되나요?", "수명은 얼마나 되나요?" 등입니다. 이 FAQ는 Schema.org의 FAQPage 스키마로 마크업되어 구글 검색 결과에서 직접 펼쳐져 보일 수 있습니다 — 검색 결과에서 차지하는 영역이 넓어져 클릭률이 올라갑니다.

마지막으로 최종 CTA 섹션에서 "지금 바로 무료 상담 예약"과 비용 안내 페이지 링크를 제공합니다.

## 3.3 의사 프로필 — 환자가 가장 궁금해하는 "이 사람"

환자에게 가장 중요한 결정 요인 중 하나가 "누가 나를 치료하는가"입니다. 특히 임플란트나 교정처럼 장기간 관계를 맺는 치료에서는 의사에 대한 신뢰가 병원 선택을 좌우합니다.

의사 프로필 페이지의 필수 요소는 다음과 같습니다. 고화질 전문 촬영 사진(이것이 없으면 모든 것이 무너집니다 — 핸드폰으로 찍은 사진은 절대 안 됩니다), 이름과 직함("문석준 대표원장"), 학력("서울대 치의학과, 서울대 치의학대학원 석사"), 자격증 및 전문의("통합치의학과 전문의"), 주요 경력(연도순), 전문 분야(임플란트, 심미보철 등), 소속 학회와 논문 실적, 소개 영상(유튜브 임베드 — 2분 이내로 자신의 진료 철학을 말하는 영상), 실제 진료 모습 사진(환자와 대화하거나 장비를 조작하는 모습), 담당 치료 링크(이 의사가 전문으로 하는 치료 페이지로 바로 이동), 그리고 인간적 스토리("왜 치과의사가 되었는가", "환자에게 하고 싶은 말")까지.

인간적 스토리가 중요한 이유는, 환자가 단순히 "실력 있는 의사"가 아니라 "나를 이해해줄 것 같은 의사"를 찾기 때문입니다. "치과가 무서운 마음, 저도 압니다. 어릴 때 치과에서 겁먹었던 기억이 있거든요."라는 한 마디가, 학력이나 경력보다 더 강한 신뢰를 만드는 경우가 많습니다.

## 3.4 지역 SEO 페이지 — 검색 유입의 숨은 병기

지역 SEO 페이지는 환자가 직접 보기보다는, 검색엔진을 통해 유입되게 하는 전략 페이지입니다. "아산 치과", "세종 임플란트" 같은 키워드로 검색했을 때, 메인 페이지가 아닌 해당 지역 전용 랜딩 페이지가 뜨면 환자의 관련성 인식이 높아집니다.

각 지역 페이지의 구조는 동일하지만, 내용은 지역별로 완전히 다릅니다. 히어로에서 "{지역}에서 서울비디치과까지 차로 {N}분"이라는 구체적 소요시간을 제시하고, 왜 이 병원인가 섹션에서 차별점 6~8개를 나열하고, 대표 치료 4~6개를 소개하고, 교통 안내 섹션에서 해당 지역에서 오는 구체적인 경로(자가용 루트, 시외버스, KTX 등)를 안내하고, 실제 해당 지역에서 온 환자의 리뷰를 보여주고("{지역}에서 왔는데 정말 잘 해주셨어요"), 구글/네이버 맵을 임베드하고, 마지막으로 예약 CTA를 배치합니다.

## 3.5 예약/상담 페이지 — 전환의 최종 관문

예약 페이지의 설계 원칙은 "최소한의 마찰로 최대한의 전환을"입니다. 먼저 빠른 연락 수단 3종(전화, 카카오톡, 네이버 예약)을 페이지 최상단에 크게 배치합니다. 환자의 절반 이상이 온라인 폼보다 전화나 카카오톡을 선호한다는 데이터에 기반합니다.

온라인 예약 폼은 단계별로 설계합니다. Step 1에서 치료 종류(임플란트/교정/일반진료/검진/기타)를 선택하고, Step 2에서 이름, 연락처, 희망 날짜/시간대, 특이사항을 입력하고, Step 3에서 확인 후 제출합니다. 한 화면에 모든 필드를 보여주는 것보다, 단계별로 나누는 것이 완료율이 2~3배 높습니다.

진료시간 정보(요일별 시간 + 공휴일)는 반드시 이 페이지에 포함시킵니다. 환자가 "지금 전화해도 되나?"를 확인하기 위해서입니다.

## 3.6 비용 안내 페이지 — 투명성이 곧 신뢰

비용 안내는 대부분의 병원이 꺼리지만, 공개하는 것이 무조건 유리합니다. 이유는 세 가지입니다. 첫째, 환자의 90%가 비용을 가장 먼저 궁금해합니다. 둘째, 비용을 숨기면 "비싸니까 숨기는 거겠지"라는 의심을 삽니다. 셋째, 의료법 시행규칙 제42조의2에 따라 비급여 진료비용은 홈페이지에 공개해야 합니다 — 법적 의무이기도 합니다.

가격은 정확한 금액이 아닌 범위로 표시합니다(예: 임플란트 80~150만원). 보험 적용 여부를 구분하고, "정확한 비용은 진단 후 상담을 통해 안내해 드립니다"라는 법적 고지를 포함합니다. 카드 결제와 할부 옵션도 안내합니다. 치료 옵션 간 비교표(예: 금인레이 vs 지르코니아 인레이 vs 레진)를 제공하면 환자의 이해도가 높아지고, 상담 시 설명 시간이 줄어듭니다.

## 3.7 비포/애프터 갤러리 — 실력의 증거

"백 마디 말보다 한 장의 사진"입니다. 치료 전후 사진은 환자에게 가장 강력한 설득 수단입니다. 갤러리 페이지에는 카테고리 필터(임플란트/교정/심미/미백/잇몸), 좌우 드래그 비교 슬라이더, 케이스 상세 정보(치료 기간, 담당 의사, 간략한 설명), 담당 의사 프로필 링크, "총 N건의 치료 사례" 통계를 포함합니다.

의료법 주의사항이 있습니다. 환자 서면 동의는 법적 필수이며, 동일 조건(조명, 각도)에서 촬영해야 합니다. "개인에 따라 결과가 다를 수 있습니다"라는 고지를 반드시 포함하고, 과장이나 허위 사례는 절대 금지입니다.

# 4. UI/UX 설계 기준

## 4.1 디자인 시스템 — 일관성이 전문성을 만든다

병원 홈페이지의 디자인은 "예쁘게"가 아니라 "신뢰감 있게"가 핵심입니다. 화려한 애니메이션이나 트렌디한 레이아웃보다, 깔끔하고 정돈되고 읽기 쉬운 디자인이 의료기관에 훨씬 적합합니다. 환자는 "여기 사이트가 멋지다!"가 아니라 "여기 정보가 정확하고 신뢰할 수 있겠다"라고 느끼기를 원합니다.

**컬러 팔레트**는 6가지 역할로 나눕니다. 프라이머리 컬러는 병원의 브랜드 아이덴티티(서울비디치과는 골드 #C9A962 — 프리미엄 느낌), 세컨더리 컬러는 보조/링크색(블루 #0369a1), 배경색은 다크/라이트 모드(#0a0908 / #FFFFFF), 텍스트 컬러는 메인(#1a1a1a)/서브(#666666)로 가독성 확보, CTA 컬러는 프라이머리와 동일하게 골드로 눈에 띄게, 상태 컬러는 성공(녹)/경고(노)/에러(적)입니다. 컬러 선택의 원칙은 "차분하되 한 가지 포인트 컬러로 고급감을 준다"입니다. 치과는 골드/네이비, 피부과는 화이트/핑크, 한의원은 브라운/그린이 잘 맞습니다.

**타이포그래피**는 한글 폰트로 Pretendard(CDN 로딩)를 사용합니다. 시스템 폰트와 달리 모든 기기에서 동일한 렌더링을 보장하며, 가독성이 뛰어납니다. H1은 Bold 36~48px(페이지당 1개만), H2는 Bold 28~36px(섹션 타이틀), H3는 Semi-Bold 20~24px(서브 타이틀), 본문은 Regular 15~16px에 line-height 1.7(줄 간격이 넉넉해야 긴 글도 편하게 읽힙니다)로 설정합니다.

**아이콘**은 Font Awesome 6.4를 CDN으로 로딩합니다. 수천 개의 아이콘을 즉시 사용할 수 있고, 별도 파일을 관리할 필요가 없습니다. 로고와 같은 고유 아이콘만 커스텀 SVG로 제작합니다.

## 4.2 반응형 — 모바일이 75%, 무조건 모바일 퍼스트

2026년 현재 병원 홈페이지 트래픽의 75% 이상이 모바일에서 발생합니다. 반응형 브레이크포인트는 모바일(~767px, 1열 레이아웃 + 하단 고정 CTA), 태블릿(768~1023px, 2열), 데스크톱(1024px~, 3~4열 + 플로팅 CTA) 3단계로 설정합니다.

모바일 퍼스트 필수 요소를 나열합니다. 하단 고정 CTA 바(전화/카카오/예약/길찾기 4버튼)는 모바일에서 전환율을 직접 좌우합니다. 터치 타겟은 최소 44×44px — 이보다 작으면 손가락 큰 사람은 옆 버튼을 누르게 됩니다. 전화번호는 반드시 `tel:` 링크로 — 터치하면 바로 전화가 걸려야 합니다. 폰트는 최소 14px, 권장 16px — 작은 글씨는 환자(특히 중장년층)의 이탈을 부릅니다. 이미지는 WebP 포맷에 lazy loading 적용 — 모바일 데이터 환경에서 빠르게 로딩되어야 합니다. 스크롤 애니메이션은 과도하지 않게 — 멋있어 보이지만 성능을 떨어뜨립니다.

## 4.3 접근성 — 법적 의무이자 윤리적 책임

모든 인터랙티브 요소에 ARIA 라벨을 부여합니다. 키보드만으로도 전체 사이트를 탐색할 수 있도록 Tab 순서를 보장합니다(시맨틱 HTML 사용). 색상 대비는 WCAG 2.1 AA 기준 4.5:1 이상 — 시력이 약한 분도 텍스트를 읽을 수 있어야 합니다. 모든 이미지에 alt 텍스트를 기입합니다(스크린 리더 대응 + SEO 효과). 키보드 포커스 시 시각적 표시(:focus-visible)를 제공하고, 페이지 상단에 "본문 바로가기" 건너뛰기 링크를 넣습니다.

# 5. SEO 인프라

## 5.1 온페이지 SEO — 모든 페이지의 DNA

SEO는 "검색엔진 최적화"이지만, 본질은 "환자가 검색했을 때 우리 병원이 나오게 하는 것"입니다. 80개 페이지 모든 곳에 동일한 SEO 인프라가 내장되어야 합니다.

**HTML Head 필수 태그 14종**을 하나씩 설명합니다.

첫째, `<title>` 태그는 검색 결과에서 가장 크게 보이는 텍스트입니다. 60자 이내로 "{페이지 제목} | {병원명}" 형식을 씁니다. "임플란트 | 서울비디치과"처럼 해당 페이지의 핵심 키워드와 병원명을 동시에 담습니다. 80개 페이지 모두 고유한 타이틀이어야 합니다 — 중복되면 구글이 해당 페이지를 저품질로 판단합니다.

둘째, `<meta description>` 은 검색 결과에서 타이틀 아래에 표시되는 설명문입니다. 150자 이내로, 환자가 "이 페이지를 클릭해야 하는 이유"를 담습니다. "서울비디치과 임플란트 — 서울대 15인 원장 협진, 네비게이션 정밀 수술, 수면 임플란트 가능. 비용 안내 및 무료 상담 예약."처럼 키워드와 행동 유도를 함께 넣습니다.

셋째, `<link rel="canonical">`은 해당 페이지의 정규 URL을 선언합니다. 같은 콘텐츠가 여러 URL로 접근 가능할 때(http/https, www 유무, 쿼리 파라미터 등) 검색엔진에 "이것이 원본 URL"임을 알려줍니다.

넷째, `<meta name="robots">`로 크롤링 및 색인 규칙을 지정합니다. "index, follow, max-image-preview:large, max-snippet:-1, max-video-preview:-1"이 표준 설정입니다.

다섯째부터 일곱째, Geo 태그 3종(geo.region, geo.placename, geo.position)은 지역 SEO의 핵심입니다. "KR-44"(충남), "천안시, 충청남도", "36.8151;127.1139"처럼 병원의 물리적 위치를 명시합니다.

여덟째, AI 검색 최적화 태그(ai.summary)는 2026년 현재 ChatGPT, Gemini, Perplexity 등 AI 검색 엔진이 참조하는 메타 정보입니다. 200자 이내로 해당 페이지의 핵심 정보를 요약합니다.

아홉째부터 열한째, Open Graph 태그 6종(og:type, og:site_name, og:title, og:description, og:url, og:image)은 카카오톡, 페이스북, 라인 등에서 URL을 공유할 때 표시되는 미리보기를 결정합니다. og:image는 1200×630px이 표준이며, 병원 로고와 대표 이미지가 잘 보이는 디자인이어야 합니다.

열두째부터 열세째, Twitter Card 태그(twitter:card, twitter:title, twitter:description, twitter:image)는 트위터/X에서의 미리보기를 결정합니다.

열네째, 검색엔진 인증 메타태그(google-site-verification, naver-site-verification)는 해당 도메인의 소유권을 증명합니다.

추가로 H1 태그는 페이지당 정확히 1개만, H2~H3는 계층적으로 사용합니다. 모든 이미지에 alt 태그를 달고, 내부 링크를 적극적으로 사용하며(치료 페이지에서 담당 의사 페이지로, 의사 페이지에서 관련 치료 페이지로), HTML lang 속성을 ko로 설정합니다.

## 5.2 기술적 SEO — 검색엔진이 사이트를 이해하는 방법

**sitemap.xml**은 사이트의 모든 URL을 검색엔진에 알려주는 지도입니다. 각 URL에 lastmod(마지막 수정일), changefreq(변경 빈도), priority(중요도)를 지정합니다. 메인 페이지는 priority 1.0, 치료 페이지는 0.9, 의사 페이지는 0.8, 지역 페이지는 0.7, 콘텐츠 페이지는 0.6, 법적/기타 페이지는 0.3으로 설정합니다.

**robots.txt**는 크롤링 허용/차단 규칙을 정의합니다. 관리자 페이지(/admin/), 회원 페이지(/auth/), API 경로(/api/), 워커 파일(/_worker.js)은 차단하고, 나머지는 모두 허용합니다. 사이트맵 경로를 명시합니다.

**IndexNow 키 파일**은 Bing과 네이버에 새 페이지를 즉시 알려주는 메커니즘입니다. 새 콘텐츠를 올리면 API 호출 한 번으로 수분 내에 색인이 가능합니다 — 기존의 "크롤러가 올 때까지 며칠 기다리기"를 건너뛸 수 있습니다.

**URL 설계 원칙**은 영문 소문자, 하이픈 구분, .html 확장자 제거(Clean URL), 짧고 명확하게, 계층적 구조, 한글 사용 금지입니다. "/treatments/implant"는 좋은 URL이고, "/Treatments/Implant.html"이나 "/치료/임플란트"는 나쁜 URL입니다. Clean URL은 서버 라우팅으로 구현합니다 — 실제 파일은 treatments/implant.html이지만, 사용자에게 보이는 URL은 /treatments/implant입니다.

# 6. 구조화 데이터 (Schema.org)

## 6.1 구조화 데이터가 왜 중요한가

구조화 데이터는 검색엔진에게 "이 페이지가 무엇에 관한 것인지"를 기계가 이해할 수 있는 형태로 알려주는 코드입니다. JSON-LD 형식으로 HTML에 삽입하며, 검색 결과에서 리치스니펫(별점, FAQ 펼침, 가격 범위, 치료 과정 등)으로 표시됩니다.

서울비디치과는 80개 페이지에 134개의 JSON-LD 항목과 30종 이상의 스키마 유형을 사용합니다. 이것이 과도해 보일 수 있지만, 검색 결과에서 눈에 띄는 병원과 그렇지 않은 병원의 차이가 바로 이 구조화 데이터입니다.

## 6.2 페이지별 필수 스키마

**모든 페이지 공통으로** 4가지 스키마가 들어갑니다. Dentist(또는 MedicalBusiness)는 병원의 기본 정보(이름, 주소, 전화, 영업시간, 평점, 서비스 지역)를 담으며, 구글 지식패널과 지도 노출에 직접적으로 영향을 줍니다. BreadcrumbList는 "홈 > 진료안내 > 임플란트" 같은 경로를 표시하여 사이트의 계층 구조를 검색 결과에 보여줍니다. WebSite + SearchAction은 구글 검색 결과에 사이트 내 검색창을 표시하게 합니다. SpeakableSpecification은 구글 어시스턴트, 시리 같은 음성 검색에 대응합니다.

**치료 페이지 전용으로** 8가지 스키마를 추가합니다. MedicalWebPage와 MedicalProcedure는 이 페이지가 의료 정보임을 선언합니다. MedicalTherapy는 치료법의 적응증과 금기사항을 포함합니다. HowTo + HowToStep은 치료 과정을 단계별로 보여주는 리치스니펫을 생성합니다. FAQPage는 FAQ를 검색 결과에서 직접 펼쳐 보여줍니다. AggregateOffer는 가격 범위를 표시하고, AggregateRating은 별점을 표시합니다. Product 스키마는 치료 서비스를 하나의 제품으로 마크업하여 구글 쇼핑과 유사한 정보 패널을 생성합니다.

**의사 페이지 전용으로** Person(기본 인적 정보), EducationalOccupationalCredential(학력/자격), CollegeOrUniversity(출신 대학), Organization(소속 학회) 스키마를 사용합니다.

**지역 페이지 전용으로** Dentist + areaServed(서비스 권역), City(대상 도시), OpeningHoursSpecification(영업시간)을 사용합니다.

## 6.3 Dentist 스키마 상세

Dentist 스키마는 가장 중요한 구조화 데이터입니다. 여기에 병원명, 영문명, URL, 전화번호, 대표 이미지, 로고, 주소(PostalAddress), 좌표(GeoCoordinates), 영업시간(OpeningHoursSpecification — 요일별로), 평점(AggregateRating), 외부 프로필 링크(sameAs — 네이버 플레이스, 유튜브, 카카오톡), 서비스 지역(areaServed — City 목록), 지도 링크(hasMap), 가격대(priceRange)를 모두 담습니다.

규모별 적정 JSON-LD 항목 수는 소규모(20페이지) 40~60개, 중규모(80페이지) 130~150개, 대규모(200페이지+) 300개 이상입니다.

# 7. 마케팅 & 전환 인프라

## 7.1 필수 마케팅 도구 — 데이터 없이는 개선도 없다

홈페이지를 만들어 놓기만 하면 환자가 알아서 올까요? 절대 아닙니다. 환자가 어디서 오고, 어떤 페이지를 보고, 어디서 이탈하는지를 데이터로 알아야 개선할 수 있습니다.

**Google Tag Manager(GTM)**는 모든 추적 코드를 한 곳에서 관리하는 컨테이너입니다. HTML 코드를 수정하지 않고도 구글 애널리틱스, 광고 추적, 이벤트 추적 등을 추가/수정/삭제할 수 있습니다. `<head>`와 `<body>`에 각각 스니펫을 삽입하면 됩니다.

**Google Analytics 4(GA4)**는 트래픽 분석의 표준입니다. 방문자 수, 유입 경로, 페이지별 체류시간, 이탈률, 전환율 등을 측정합니다. GTM을 통해 관리하면 설치/수정이 편합니다.

**Amplitude**는 GA4보다 정밀한 사용자 행동 분석 도구입니다. "환자가 어떤 순서로 페이지를 이동하는가?", "예약을 시작했다가 어느 단계에서 포기하는가?" 같은 깊은 질문에 답합니다.

**Google Search Console**은 구글 검색에서의 성능(노출수, 클릭수, 평균 순위, CTR)을 모니터링합니다. **Naver Search Advisor**는 네이버에서 동일한 역할을 합니다. **Bing Webmaster**는 Bing 검색 + IndexNow 즉시 색인 기능을 제공합니다.

## 7.2 이벤트 트래킹 — 환자의 행동을 읽는 9가지 신호

병원 홈페이지에서 추적해야 할 핵심 이벤트 9가지가 있습니다. 이 이벤트들은 환자의 전환 여정을 데이터로 보여줍니다.

reservation_click(예약 버튼 클릭)은 가장 중요한 전환 이벤트입니다. 어느 페이지에서, 어떤 소스(네이버/카카오/온라인)의 예약을 클릭했는지 추적합니다. phone_click(전화번호 클릭)은 모바일에서 전환율이 가장 높은 이벤트입니다. kakao_click(카카오톡 버튼 클릭)은 한국 시장에서 전화 다음으로 전환율이 높습니다. map_click(지도/길찾기 클릭)은 방문 의사가 확인된 환자의 신호입니다. treatment_view(치료 페이지 조회)는 어떤 치료에 관심이 높은지, 체류시간은 얼마인지를 보여줍니다. doctor_view(의사 페이지 조회)는 어떤 의사가 환자의 관심을 끄는지 알려줍니다. pricing_view(비용 페이지 조회)는 체류시간이 길수록 구매 의향이 높은 환자입니다. scroll_depth(스크롤 25/50/75/100%)는 콘텐츠가 실제로 읽히고 있는지를 보여줍니다. cta_click(CTA 버튼 클릭)은 어떤 유형의 CTA가, 어떤 위치에서 가장 효과적인지 분석합니다.

## 7.3 전환 최적화 — CTA의 우선순위

CTA 버튼의 우선순위는 전화 → 카카오톡 → 네이버 예약 → 온라인 폼 → 길찾기 순입니다. 전화가 1순위인 이유는 즉각적인 반응이 가능하고, 긴급 환자(치통, 치아 파절)의 전환율이 가장 높기 때문입니다. 카카오톡이 2순위인 이유는 한국 사용자가 전화보다 메시지를 부담 없이 느끼기 때문입니다. 네이버 예약이 3순위인 이유는 네이버 리뷰와 연동되어 후속 리뷰 유도가 가능하기 때문입니다.

**구글 비즈니스 프로필 연동**은 강력한 신뢰 수단입니다. Google Places API를 통해 실시간 리뷰를 홈페이지에 로딩하고(/api/google-reviews), 메인 페이지에 별점과 리뷰 수를 표시하고, "구글 리뷰 남기기" 버튼으로 신규 리뷰를 유도합니다.

## 7.4 콘텐츠 시스템 — 살아있는 사이트를 만드는 방법

병원이 직접 관리하는 콘텐츠(블로그, 유튜브)를 홈페이지에 자동으로 반영하는 시스템입니다. InBlog나 네이버 블로그의 RSS를 프록시하여(/api/inblog-rss) 칼럼 페이지에 최신 글을 표시하고, YouTube RSS를 프록시하여(/api/youtube-rss) 영상 페이지에 최신 영상을 표시합니다. 이렇게 하면 병원은 평소 하던 대로 블로그를 쓰고 유튜브를 올리면 홈페이지가 자동으로 업데이트됩니다.

# 8. 다국어 지원

## 8.1 왜 다국어가 필요한가

한국에 거주하는 외국인은 250만 명을 넘어섰고, 특히 산업단지가 있는 지역(천안, 아산, 화성 등)에서는 외국인 환자 비율이 높습니다. 베트남, 중국, CIS(러시아어권), 캄보디아, 네팔 근로자가 많고, 미군 기지 주변에는 영어 사용자가 많습니다. 이들이 치과를 검색할 때 모국어로 된 정보를 제공하는 병원은 거의 없습니다 — 그 자체가 압도적 차별화입니다.

## 8.2 Weglot — 1줄의 코드로 5개 언어

Weglot은 JavaScript 1줄만 삽입하면 전체 사이트를 자동 번역해주는 서비스입니다. URL 구조가 서브디렉토리 방식(/en/, /vi/, /zh/)으로 자동 생성되고, 언어 선택기가 자동으로 나타나며, Visual Editor로 번역을 수동 수정할 수 있습니다.

서울비디치과의 언어 전략을 예로 들면, 영어(미군, 국제 학생, 의료관광), 베트남어(산업단지 최다 외국인 근로자), 중국어 간체(근로자, 유학생), 러시아어(우즈베키스탄, 카자흐스탄, 키르기스스탄 등 CIS 전체를 하나의 언어로 커버), 일본어(의료관광 고객 — 1인당 매출이 가장 높은 세그먼트) 5개 언어로 구성합니다. Pro 플랜(월 EUR 79)으로 5개 언어, 200,000 단어까지 커버됩니다.

**용어집 설정**이 중요합니다. 병원명, 자체 브랜드 치료명(예: 글로우네이트 → Glownate), "인비절라인 → Invisalign", 전화번호(번역하면 안 됨), 주소(로마자 표기로 지정)를 용어집에 등록하면 자동 번역 시에도 이 단어들은 지정된 형태로 유지됩니다.

구현 코드는 모든 HTML 파일의 `</head>` 직전에 Weglot CDN 스크립트와 초기화 코드 2줄만 넣으면 됩니다. 추가 코드 작업은 필요 없습니다.

# 9. 기술 스택 & 아키텍처

## 9.1 왜 Hono + Cloudflare Pages인가

병원 홈페이지의 기술 스택을 선택할 때 가장 중요한 기준은 "빠르고, 안정적이고, 관리 비용이 거의 없는 것"입니다. 워드프레스는 플러그인 충돌, 보안 취약점, 서버 관리라는 만성적 문제가 있습니다. Wix나 Squarespace는 커스터마이징에 한계가 있고, SEO 최적화가 어렵습니다.

Hono는 Cloudflare Workers에서 동작하는 경량 웹 프레임워크로, Express.js와 비슷한 문법이지만 번들 크기가 12KB 미만입니다. Cloudflare Pages는 전 세계 300개 이상의 데이터 센터에서 콘텐츠를 서비스하는 CDN이며, 무료 플랜으로 무제한 요청과 무제한 대역폭을 제공합니다. Vite는 빌드 도구로, TypeScript를 빌드하고 정적 파일을 최적화합니다.

이 조합의 장점은 월 호스팅 비용 $0(무료 플랜), 글로벌 CDN으로 세계 어디서든 빠른 로딩, 서버리스이므로 서버 관리 불필요, DDoS 방어 및 SSL 인증서 자동 제공, 배포는 `npm run deploy` 한 줄입니다.

## 9.2 프로젝트 구조 상세

프로젝트의 뼈대는 다음과 같습니다. `src/index.tsx`가 Hono 백엔드 진입점으로, API 라우트와 HTML 서빙 라우트를 모두 담당합니다. `public/` 디렉토리 아래에 모든 정적 파일(HTML, CSS, JS, 이미지)이 위치합니다. HTML 파일은 역할별로 하위 폴더(treatments/, doctors/, area/, cases/, column/, video/, notice/, auth/, admin/)로 분류합니다. CSS는 단일 파일(site-v5.css)로 통합하여 요청 수를 최소화합니다. JavaScript는 기능별(main.js, gnb.js, analytics.js, gallery.js, column.js, video.js)로 분리하여 유지보수성을 높입니다. 이미지는 용도별(doctors/, facility/, icons/)로 분류합니다.

## 9.3 백엔드 API 라우트

API는 8개 라우트로 구성됩니다. `/api/health`는 서버 상태 확인용, `/api/google-reviews`는 Google Places API를 프록시하여 리뷰를 안전하게 로딩(API 키가 프론트엔드에 노출되지 않음), `/api/inblog-rss`와 `/api/youtube-rss`는 각각 블로그와 유튜브 RSS를 프록시, `/api/chat`은 선택적 AI 챗봇, `/api/indexnow`는 검색엔진 즉시 색인 요청, `/api/google-ping`은 구글에 사이트맵 갱신을 알리는 역할입니다.

## 9.4 호스팅 비용 — 왜 $0인가

Cloudflare Pages 무료 플랜은 무제한 요청, 무제한 대역폭, 월 500회 빌드를 제공합니다. 80페이지 규모의 병원 홈페이지는 이 한도 안에서 충분히 운영됩니다. Pro 플랜($20/월)은 빌드 횟수가 5,000회로 늘어나고, Workers 요청이 10M/월로 확장되지만, 대부분의 병원은 무료 플랜으로 충분합니다. 도메인 비용(연 2~3만원)과 Weglot(다국어, 월 EUR 79)이 사실상 유일한 운영 비용입니다.

# 10. 검색엔진 등록 & 색인

## 10.1 Google 등록 절차

홈페이지를 만들었으면 검색엔진에 "우리가 여기 있습니다"라고 알려야 합니다. 아무리 멋진 사이트를 만들어도 검색 결과에 나오지 않으면 의미가 없습니다.

Google Search Console에서 HTML 메타태그로 소유권을 인증하고, 사이트맵을 제출하고, 메인 페이지 URL을 검사하여 색인 생성을 요청합니다. 색인이 완료되면 구조화 데이터가 정상적으로 인식되는지 '향상' 메뉴에서 확인합니다. 보통 메인 페이지는 1~3일, 치료 페이지는 3~7일, 전체 사이트는 2~4주 안에 색인됩니다.

## 10.2 Naver 등록 절차

한국에서는 네이버가 여전히 검색 점유율의 상당 부분을 차지하므로, Naver Search Advisor 등록은 필수입니다. HTML 메타태그로 인증하고, 사이트맵을 제출하고, RSS(블로그 연동 시)를 제출하고, 핵심 URL을 수동으로 수집 요청합니다. robots.txt 검증도 수행합니다. 네이버의 색인은 구글보다 느려 3~14일이 걸릴 수 있습니다.

## 10.3 Bing 등록 및 IndexNow

Bing Webmaster는 Google Search Console과 연동하여 간편하게 등록할 수 있습니다. 핵심은 IndexNow입니다. IndexNow 키 파일을 배포하고, /api/indexnow 엔드포인트를 통해 사이트맵의 모든 URL(73개)을 한 번에 전송하면, Bing은 물론 Naver와 Yandex에도 즉시 색인 요청이 전달됩니다. Bing의 IndexNow 덕분에 새 페이지가 수분~수시간 내에 색인될 수 있습니다.

## 10.4 등록 후 모니터링

메인 페이지 색인은 1~3일 내 확인, 치료 페이지 색인은 3~7일 내 사이트맵 커버리지 확인, 구조화 데이터 오류는 7일 내 GSC 향상 메뉴에서 0 에러 확인, 네이버 수집은 3~14일 내 Search Advisor에서 현황 확인, Bing 색인은 1~3일 내 확인합니다.

# 11. 법적 필수 요소

## 11.1 의료광고 규정 — 한 줄이라도 잘못되면 처벌

의료법 56조는 의료광고의 범위와 금지사항을 규정합니다. 병원 홈페이지의 모든 텍스트와 이미지는 의료광고에 해당하며, 다음 사항을 반드시 준수해야 합니다.

광고 내용 사전 심의 또는 면제 확인이 필요합니다. 치료 전후 사진은 동일 조건(조명, 각도)에서 촬영해야 하고, 반드시 환자의 서면 동의를 받아야 합니다. 비급여 가격은 홈페이지에 공개해야 합니다(의료법 시행규칙 42조의2). "최고", "100%", "완벽" 등 과장 표현은 금지되며, 전후 비교 시 "개인에 따라 결과가 다를 수 있습니다"라는 고지가 필수입니다. 의사의 경력은 정확하게 기재해야 하며, 허위 경력은 법적 처벌 대상입니다.

모든 치료 페이지 하단에 다음 문구를 포함합니다: "본 홈페이지의 모든 의료 정보는 의료법 및 보건복지부 의료광고 가이드라인을 준수하여 제공하고 있으며, 특정 개인의 결과는 개인에 따라 달라질 수 있습니다."

## 11.2 개인정보보호 — 예약 폼 하나에도 법이 적용된다

개인정보보호법 30조에 따라 개인정보처리방침을 홈페이지에 게시하는 것은 법적 의무입니다. 예약 폼이나 회원가입에서 개인정보를 수집할 때는 15조에 따라 수집 동의를 받아야 합니다. 마케팅 수신 동의는 정보통신망법 50조에 따라 별도의 선택 동의를 받습니다. 다국어 사이트의 경우 해외 접속자를 위한 GDPR 쿠키 배너도 고려합니다.

개인정보처리방침에 반드시 포함해야 할 10가지 항목은 수집하는 개인정보 항목, 수집·이용 목적, 보유·이용 기간, 제3자 제공 여부, 처리 위탁, 정보주체의 권리·의무, 파기 절차, 보호책임자, 자동 수집 장치(쿠키), 권익침해 구제방법입니다.

## 11.3 기타 법적 요소

푸터에 사업자 정보(상호, 대표자, 사업자등록번호, 주소), 의료기관 정보(의료기관명, 대표원장명), 저작권 고지("© 2026 {병원명}")를 표시합니다. 치료 페이지에는 면책 고지("개인에 따라 다를 수 있음")를 포함합니다.

# 12. 성능 & 보안

## 12.1 Core Web Vitals — 구글이 측정하는 사이트 품질

구글은 2021년부터 Core Web Vitals를 검색 순위에 반영하고 있습니다. 세 가지 지표가 핵심입니다.

**LCP(Largest Contentful Paint)**는 페이지에서 가장 큰 콘텐츠가 보이기까지의 시간으로, 2.5초 미만이 목표입니다. 히어로 이미지를 WebP로 최적화하고, preload를 적용하면 달성 가능합니다. **FID(First Input Delay)**는 사용자의 첫 번째 입력(버튼 클릭 등)에 대한 응답 시간으로, 100ms 미만이 목표입니다. JavaScript를 비동기(async/defer)로 로딩하면 달성됩니다. **CLS(Cumulative Layout Shift)**는 페이지 로딩 중 레이아웃이 얼마나 흔들리는지를 나타내며, 0.1 미만이 목표입니다. 이미지와 광고에 명시적 크기를 지정하고, 폰트 로딩을 최적화하면 됩니다.

성능 최적화 체크리스트는 이미지 최적화(WebP + lazy loading), CSS/JS 최소화(Vite 빌드 자동), 폰트 최적화(preconnect + preload), CDN(Cloudflare 자동), 캐싱(Cache-Control 헤더), 크리티컬 CSS 인라인, JS 비동기 로딩, 주요 페이지 prefetch입니다.

## 12.2 보안 — Cloudflare가 해결하는 대부분의 것

HTTPS는 Cloudflare가 자동으로 처리하며, SSL 인증서도 자동 발급됩니다. DDoS 방어도 Cloudflare의 기본 기능입니다. 추가로 보안 헤더(Content-Security-Policy, X-Frame-Options)를 설정하고, API 키는 반드시 서버사이드에서만 사용하며(프론트엔드 노출 금지), 예약 폼 데이터는 서버측에서 검증합니다.

# 14. 운영 가이드

## 14.1 홈페이지는 만들고 끝이 아니다

홈페이지는 한 번 만들어 놓으면 영원히 쓸 수 있는 간판이 아닙니다. 살아있는 사이트를 유지해야 환자가 계속 찾아오고, 검색엔진이 계속 노출시킵니다.

**매일** 해야 할 일은 예약 확인과 응대입니다. **매주** 해야 할 일은 블로그/칼럼 1개 이상 발행(InBlog 또는 네이버 블로그)과 Google Business Profile 게시물 1개 작성입니다. 이 두 가지는 홈페이지의 "신선도"를 유지하는 데 결정적이며, 검색엔진은 정기적으로 새 콘텐츠가 올라오는 사이트를 "활동적인 사이트"로 평가하여 순위를 올려줍니다.

**매월** 해야 할 일은 비포/애프터 사례 2~3건 추가(환자 동의 + 사진 + 설명)와 비용 안내 페이지의 가격 변동 확인입니다. **분기별**로 SEO 성과 리포트(GSC 데이터 분석)와 Core Web Vitals 점검을 합니다. **반기별**로 개인정보처리방침을 갱신하고, **연 1회** 저작권 연도를 업데이트합니다.

## 14.2 병원에서 직접 관리 가능한 것

블로그 작성(InBlog/네이버 블로그 — 난이도 ★☆☆), 유튜브 영상 업로드(YouTube Studio — ★☆☆), 공지사항 작성(관리자 CMS — ★☆☆), 리뷰 응답(Google/Naver — ★☆☆), GBP 게시물(Google Business Profile — ★★☆), Weglot 번역 수정(Weglot Dashboard — ★★☆), 예약 확인(이메일/카카오 알림 — ★☆☆)은 병원 직원이 쉽게 할 수 있습니다.

# 15. 부록: 커스터마이징 옵션

## 15.1 추가 기능 — 홈페이지를 경쟁 무기로 만드는 선택지

기본 홈페이지가 "있어야 할 것을 갖춘 것"이라면, 추가 기능은 "경쟁 병원이 하지 않는 것"입니다.

**바이럴 퀴즈**("나의 치아 캐릭터는?")는 환자가 재미로 참여하고 결과를 카카오톡, 인스타그램으로 공유하게 만듭니다. 공유된 결과 페이지에 예약 CTA가 포함되어 자연스러운 바이럴 마케팅이 됩니다. **비용 계산기**("내 임플란트 비용 알아보기")는 간단한 조건(잔존골 상태, 임플란트 종류, 보철물 종류)을 선택하면 예상 비용 범위를 보여주는 도구로, 결과 확인을 위해 전화번호를 입력하게 하여 리드를 확보합니다. **진단 퀴즈**("나에게 맞는 치료법은?")도 유사한 리드 확보 수단입니다. **AI 챗봇**은 24시간 자동 상담으로 야간이나 주말에도 환자의 질문에 즉시 응답합니다. **카카오 알림톡**은 예약 확인, 리마인더, 내일 방문 안내를 자동 발송하여 노쇼율을 줄입니다.

## 15.2 과별 특화 요소

**치과**는 수술실 규모, 디지털 장비(CT, 네비게이션), 수면치료/무통 시스템, 소아치과 분리를 강조합니다. **피부과/성형외과**는 비포/애프터를 전면에 배치하고, 시술 가격 비교표, 1:1 상담 예약(프라이버시), SNS(인스타그램) 연동을 강화합니다. **한의원**은 한방 치료 원리 설명, 증상별 페이지(검색 유입), 체질 진단 퀴즈(리드 마그넷), 건강보험 한방 진료 안내(가격 장벽 해소)를 접목합니다.

## 15.3 가격 패키지 참고

**Basic**: 메인, 예약, 비용, 오시는 길, 치료 5개, 의사 3명 — 약 15페이지. 최소한의 온라인 존재감을 갖추는 패키지입니다.

**Standard**: Basic + 치료 15개, 전체 의사 프로필, 지역 5개, FAQ, 갤러리 — 약 40페이지. SEO와 신뢰 구축이 가능해지는 패키지입니다.

**Enterprise**: Premium + AI 챗봇, CMS, 바이럴 퀴즈, 비용 계산기, 월 유지보수 — 100페이지+ 규모의 종합 솔루션입니다.
