# 1. 프로젝트 개요

## 1.1 이 기획서의 목적

이 문서는 **병원(치과/의원/한의원) 홈페이지 제작**을 위한 완전한 기획서입니다.
서울비디치과(bdbddc.com) 실제 운영 사이트를 기준으로, 검증된 모든 기능과 구조를 정리합니다.

## 1.2 적용 대상

| 대상 | 적합도 |
|------|--------|
| 치과 | ★★★★★ (최적화됨) |
| 피부과/성형외과 | ★★★★☆ (비포/애프터 강화) |
| 내과/일반의원 | ★★★★☆ (치료 → 진료과목으로 변경) |
| 한의원 | ★★★★☆ (한방 진료 특화) |
| 안과/이비인후과 | ★★★☆☆ (구조 간소화 가능) |
| 종합병원 | ★★☆☆☆ (별도 설계 필요) |

## 1.3 핵심 철학

> **"환자가 병원을 인지하는 순간부터 지인에게 소개하기까지의 전체 여정을 설계한다"**
> — Patient Funnel (페이션트 퍼널)

### 환자 퍼널 10단계와 홈페이지 대응

| 단계 | 환자 행동 | 홈페이지 대응 |
|------|----------|--------------|
| 1. 인지 | 검색/광고로 병원 발견 | SEO, 지역 페이지, 구조화 데이터 |
| 2. 관심 | 홈페이지 방문 | 히어로 섹션, 첫인상 디자인 |
| 3. 탐색 | 치료 정보 확인 | 치료 상세 페이지 24개+ |
| 4. 비교 | 타 병원과 비교 | 차별점, 의사 프로필, 시설 |
| 5. 신뢰 | 리뷰/실력 확인 | 구글 리뷰 위젯, 비포/애프터 |
| 6. 결심 | 예약 결정 | CTA 버튼 (전화/카카오/온라인) |
| 7. 방문 | 병원 찾아옴 | 오시는 길, 층별 안내 |
| 8. 경험 | 진료 받음 | (오프라인 영역) |
| 9. 만족 | 후기 작성 | 리뷰 작성 유도 링크 |
| 10. 소개 | 지인에게 추천 | 공유 기능, 바이럴 콘텐츠 |
